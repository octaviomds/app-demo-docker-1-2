# With Docker 

This example shows how to use Docker with Next.js and deploy to multiple environments with different env values. Based on the [deployment documentation](https://nextjs.org/docs/deployment#docker-image).

---

## Getting started

Bootstrap the project using [create-next-app](https://github.com/vercel/next.js/tree/canary/packages/create-next-app) with your preferred package manager:

```bash
# npm
npx create-next-app --example with-docker-multi-env nextjs-docker-multi-env

# yarn
yarn create next-app --example with-docker-multi-env nextjs-docker-multi-env

# pnpm
pnpm create next-app --example with-docker-multi-env nextjs-docker-multi-env
```

Then fill in each sample env file with the appropriate values for each target environment:

```
.env.development.sample
.env.staging.sample
.env.production.sample
```

---

## Deployment environments

| Environment | Port | Purpose |
|-------------|------|---------|
| Development | `:3001` | Integration testing |
| Staging | `:3002` | UAT / QA testing |
| Production | `:3003` | Live — end users |

### Docker + Makefile commands

```bash
# Development
make build-development
make start-development    # → http://localhost:3001

# Staging
make build-staging
make start-staging        # → http://localhost:3002

# Production
make build-production
make start-production     # → http://localhost:3003
```

## Running locally

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser. The page hot-reloads as you edit files.

> **Note:** API routes are available at `http://localhost:3000/api/hello` — edit them in `pages/api/hello.js`.

## Project structure

```
├── app/                     # App Router (Next.js 13+)
│   ├── layout.tsx           # Root layout
│   ├── page.tsx             # Home page
│   ├── globals.css          # Global styles
│   └── favicon.ico
├── components/              # Shared React components
│   └── ui/                  # shadcn/ui primitive components
├── hooks/                   # Custom React hooks
├── lib/                     # Utility functions & helpers
├── pages/
│   └── api/
│       └── hello.js         # Example API route
├── components.json          # shadcn/ui config
├── postcss.config.mjs       # PostCSS / Tailwind config
├── Dockerfile
├── Makefile
├── .env.development.sample
├── .env.staging.sample
└── .env.production.sample
```

### Key files

#### `app/`
App Router directory (Next.js 13+). Contains `layout.tsx` (root layout), `page.tsx` (home page), and `globals.css`. All routes are defined as folders inside `app/` — each with a `page.tsx` file to be publicly accessible.

#### `components.json`
shadcn/ui configuration — defines aliases, style, Tailwind base color, and import paths used by the CLI to scaffold new UI components.

#### `postcss.config.mjs`
PostCSS configuration for Tailwind CSS. Processed automatically by Next.js at build time — no manual build step required.

#### `components/ui/`
Headless primitives generated by the shadcn/ui CLI (Button, Dialog, Input, etc.). These are plain source files you own and can customise freely.

#### `hooks/`
Custom React hooks shared across pages and components (e.g. `useMediaQuery`, `useLocalStorage`).

#### `lib/`
Framework-agnostic utility functions and third-party client initialisers (e.g. `cn()` class merger, API clients, date helpers).
